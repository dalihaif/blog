### 一、GitHub 从零开始

**GitHub 是什么？**
Git 是你电脑里的"时光机"，
GitHub 就是把时光机的存档**放到网上**。

打个比方：
Git 是你手机里的相册，
GitHub 就是**百度网盘**——
本地存一份，云端再备份一份，还能分享给别人看。

► **注册账号**
打开 github.com，点 **Sign up**：
- 用户名一旦注册不能改！
  建议用真名拼音，别用 `xiaoming123`。
- 填邮箱 → 设置密码 → 验证邮件，搞定。

► **创建第一个仓库**
点右上角 **+** → **New repository**
- 起个名字，比如 `my-blog`
- 勾选 **Add a README file**
- 点 **Create repository** → 恭喜！云端仓库建好了。

---

### 二、日常四步走

进入仓库文件夹后，每天流程：

```bash
git pull          # 第一步：拉取最新（上班第一件事）
# 干活（改文件、写东西……）
git add .          # 第三步：存档
git commit -m "今天改了什么"
git push           # 第四步：推上去（下班前）
```

**拉 → 干 → 存 → 推**，
这四步够你用到退休。💪

### 三、GitHub 卡脖子？国内替代方案

在国内用 GitHub，最大的痛点是**网速慢、经常打不开**。

以下是几个靠谱的替代方案：

► **方案一：Gitee（码云）—— 国内最推荐**
地址：<ADDRESS_REMOVED>

- 服务器在国内，速度飞快
- 中文界面，免费额度够用
- 操作跟 GitHub **一模一样**，只是换了个地址

```bash
# 添加 Gitee 远程仓库
git remote add gitee https://gitee.com/你的用户名/仓库名.git
git push gitee master
```

► **方案二：同时用 GitHub + Gitee 双保险**

```bash
# GitHub 连不上就用 Gitee
git push github master
git push gitee master
```

哪个能连就用哪个，
**两个平台都维护，永不丢失**。✅

### 四、进阶操作（办公文员也能学会）

► **分支管理——安全地"试错"**

分支就像平行宇宙。
你在主宇宙（master）上正常干活，
需要试验新方案时，开一条新分支去试，
试好了合并回来，试坏了直接扔掉——**主线完全不受影响**。

```bash
git checkout -b 新方案试验   # 创建并切换到新分支
# 在新分支上干活、提交
git checkout master        # 切回主线
git merge 新方案试验      # 试好了，合并
git branch -D 新方案试验   # 试坏了，删掉分支
```

► **冲突处理——两人改了同一个地方**

Git 会在文件里标出冲突位置：

```
<<<<<<< HEAD
这是你改的内容
=======
这是同事改的内容
>>>>>>> origin/master
```

你只需要：
1. 打开文件，判断保留哪个
2. 删掉 `<<<<<<<`、`=======`、`>>>>>>>` 这三行标记
3. 保存文件，然后正常 `git add` + `git commit`

**记住：冲突永远是人来解决的，Git 只是帮你标出来。**

### 五、养成好习惯

用了这么多年 Git，总结几条铁律：

1. **勤提交，小步走** —— 每完成一个小功能就提交一次
2. **提交说明写人话** —— `git commit -m "更新"` 跟没写一样
3. **每天上班第一件事 `git pull`** —— 不然后面冲突一大堆
4. **重要节点打 Tag** —— 定稿了、上线了，打个标签方便以后找

Git 就是来终结这种痛苦的。
