## 第一部分：认识你的博客

### 1.1 这个博客是什么？

你的博客就像一个**数字笔记本**，只不过这个笔记本住在互联网上，全世界任何人都能看到。

技术上说，它叫 **Jekyll + GitHub Pages 博客**。

别被这个名字吓到，我们把它翻译成人话：

| 术语 | 人话翻译 |
|------|-----------|
| Jekyll | 一个"把文章变成网页"的工具 |
| GitHub Pages | 一个"免费托管网页"的服务 |
| Markdown | 一种"写文章的方式"，比 Word 简单100倍 |

### 1.2 博客住在哪？

你的博客住在两个地方：

```
你的电脑上：  E:\my-web\blog\          ← 你工作的地方
互联网上：  https://dalihaif.github.io/blog/  ← 所有人看到的地方
```

每次你在电脑上写好文章，点一下"发布"，互联网上的那个版本就会自动更新。

**就像你在家写日记，写完后复印一份贴在大门口，路过的人都能看到。**

## 第二部分：第一次写文章（5分钟搞定）

### 2.1 新建一篇文章

**第一步：找到文章文件夹**

打开文件夹，进入这个目录：

```
E:\my-web\blog\_posts\
```

这个文件夹里住着你的所有文章，每一篇都是一个 `.md` 文件。

**第二步：新建一个文件**

右键 → 新建 → 文本文档 → 把文件名改成：

```
2026-06-19-我的第一篇文章.md
```

> ⚠️ 注意！文件名必须严格按照这个格式：
> `年份-月份-日期-文章标题.md`
> 日期必须是真实的，标题可以用中文。

**第三步：复制这个"文章模板"进去**

用记事本打开你刚创建的文件，把下面这段**完整复制**进去：

```markdown
---
layout: post
title: "在这里写你的文章标题"
subtitle: "在这里写副标题（可选）"
date: 2026-06-19
author: "老墨"
header-img: "img/post-bg-2015.jpg"
tags:
  - 标签1
  - 标签2
categories:
  - 分类名
---

## 开头写点什么

这里是正文，想写什么就写什么。

### 小标题

可以用 --- 画一条分隔线：

---

**加粗文字** 用两个星号包围
*斜体文字* 用一个星号包围

> 引用别人的话，用大于号

- 列个清单
- 很好用
- 就是这样

```

**第四步：把内容改成你自己的**

改这几个地方就够了：

| 改哪里 | 改成什么 | 例子 |
|--------|-----------|------|
| `title:` | 你的文章标题 | `"我的云南旅行日记"` |
| `date:` | 今天的日期 | `2026-06-19` |
| `tags:` | 文章标签（随便写） | `- 旅行`、`- 日记` |
| `categories:` | 文章分类 | `- 生活随笔` |
| 正文部分 | 你想写的内容 | 随便写！ |

**第五步：保存！**

按 `Ctrl + S`，然后关闭文件。

✅ **恭喜！你写了一篇博客文章！**

### 2.2 发布到网上（让所有人看到）

文章写好了，但它现在只住在你的电脑里，互联网上的人还看不到。

要发布，只需要做一件事：**推送到 GitHub**。

**方法一：用"一键发布脚本"（推荐）**

你博客文件夹里有一个 `启动博客.bat` 文件，双击它就可以了。

**方法二：手动发布（如果上面不行）**

打开命令行（按 `Win + R`，输入 `cmd`，回车），然后依次输入：

```bash
cd /d E:\my-web\blog
git add .
git commit -m "发布了新文章"
git push origin master
```

> 不懂这些命令没关系，你只需要知道：**每次写好文章，都要做这一步，文章才会出现在网上。**

**等3-5分钟**，打开 `https://dalihaif.github.io/blog/`，就能看到你的新文章了！

## 第三部分：让文章好看——Markdown 语法速查

Markdown 是一种"写文章的方式"，用一些简单的符号来规定格式。

**把下面这张表存下来，写文章时随时查：**

### 3.1 标题

```
# 一级标题（最大）
## 二级标题
### 三级标题
#### 四级标题（最小）
```

效果就像一本书的目录，一级比一级小。

### 3.2 文字样式

```
**这段文字会加粗**
*这段文字是斜体*
~~这段文字会划掉~~
`这段代码用等宽字体显示`
```

### 3.3 列表

**数字列表：**
```
1. 第一步
2. 第二步
3. 第三步
```

**圆点列表：**
```
- 苹果
- 香蕉
- 橙子
```

### 3.4 引用和链接

```
> 这是一段引用文字
> 就像在引用别人的话

[点击这里跳转到百度](https://www.baidu.com)
```

### 3.5 插入图片

```
![图片描述](图片的网址或者文件路径)
```

**如果想用自己电脑上的图片：**

1. 把图片复制到 `E:\my-web\blog\img\` 文件夹里
2. 在文章里写：`![我的图片](/blog/img/图片名.jpg)`

### 3.6 代码块

```
这是一行代码：`print("Hello")`

这是一段代码：
​```python

    print("Hello World")
​```
```

## 第四部分：修改博客的外观

### 4.1 改博客的名字

打开这个文件：

```
E:\my-web\blog\_config.yml
```

用记事本打开，找到这一行：

```yaml
title: 老墨的博客
```

把"老墨的博客"改成你想要的名字，保存，然后**发布**（参考第二部分）。

### 4.2 改博客的简介（侧边栏那句话）

还是在 `_config.yml` 里，找到：

```yaml
sidebar-about-description: "在平凡的岗位上，做不平凡的事"
```

改成你想说的话，保存，发布。

### 4.3 换头像

1. 准备一张正方形的照片（越大越清晰）
2. 把照片命名为 `avatar-mo.jpg`
3. 复制到 `E:\my-web\blog\img\` 文件夹，替换原来的文件
4. 发布

### 4.4 换文章顶部的大图

每篇文章顶部都有一张大图片，叫 `header-img`。

**方法一：用博客自带的图片**

在文章的开头（`header-img:` 那一行），改成这些自带图片之一：

```
img/post-bg-2015.jpg      ← 默认，星空图
img/home-bg-art.jpg        ← 艺术风格
img/bg-walle.jpg          ← 电影风格
img/contact-bg.jpg        ← 简洁风格
```

**方法二：用你自己的图片**

1. 把图片放到 `E:\my-web\blog\img\` 文件夹
2. 在文章开头写：`header-img: "img/你的图片名.jpg"`
3. 保存发布

## 第五部分：修改博客的"模块"

所谓"模块"，就是博客上那些**独立的功能区块**，比如"最近文章"、"标签云"、"评论区"等等。

### 5.1 开启或关闭评论区

你的博客用的是一个叫 **Giscus** 的评论系统（基于 GitHub Discussions，免费）。

**要开启评论：**

打开 `_config.yml`，找到：

```yaml
giscus_enable: true
```

如果是 `false`，改成 `true`，保存发布。

**要关闭某篇文章的评论：**

在文章的开头（三横线之间）加上：

```yaml
giscus_enable: false
```

### 5.2 修改首页显示的文章数量

打开 `_config.yml`，找到：

```yaml
paginate: 10
```

这个数字表示"每页显示多少篇文章"。改成你想要的数字，保存发布。

### 5.3 添加"友情链接"（侧边栏）

打开 `_config.yml`，找到 `friends:` 这一段：

```yaml
friends:
  [
    { title: "GitHub", href: "https://github.com/dalihaif" },
  ]
```

想加更多链接，照着这个格式加就行：

```yaml
friends:
  [
    { title: "GitHub", href: "https://github.com/dalihaif" },
    { title: "百度", href: "https://www.baidu.com" },
    { title: "我的另一个网站", href: "这里填网址" },
  ]
```

### 5.4 修改导航栏（顶部的菜单）

导航栏的内容在 `E:\my-web\blog\_includes\header.html` 文件里。

用记事本打开，找到类似这样的内容：

```html
<li>
  <a href="{{ site.baseurl }}/">Home</a>
</li>
<li>
  <a href="{{ site.baseurl }}/archive">Archive</a>
</li>
```

想加一个新菜单项，照着格式加：

```html
<li>
  <a href="这里填链接">这里填菜单名字</a>
</li>
```

## 第六部分：在本地预览（发布前先看看效果）

有时候你写好了文章，想先看看效果再发布，可以这样做：

### 6.1 安装必要工具（只需要做一次）

1. 安装 [Ruby](https://rubyinstaller.org/)（选带 DevKit 的版本）
2. 安装 Jekyll：打开命令行，输入 `gem install jekyll bundler`

### 6.2 启动本地预览

1. 打开命令行，进入博客目录：`cd /d E:\my-web\blog`
2. 输入：`jekyll serve`
3. 等它跑完，打开浏览器，访问：`http://localhost:4000/blog/`
4. 你就能看到和线上一模一样的博客了！

> 💡 修改文章后，刷新浏览器就能看到变化，不需要重启。

## 第七部分：常见问题（踩坑指南）

### Q1：文章发布了，但网上看不到？

**可能原因1：** 还没等到。GitHub Pages 需要 3-5 分钟来处理。

**可能原因2：** 文件名格式错了。必须是 `年份-月份-日期-标题.md`，日期必须是真实日期（不能是未来）。

**可能原因3：** 没有 `git push`。写了文章要记得推送！

### Q2：中文显示乱码？

用 **记事本** 打开文件，点"文件"→"另存为"，在"编码"那里选 **UTF-8**，保存。

### Q3：图片显示不出来？

检查图片路径是否正确：
- 如果图片在 `img/` 文件夹里，路径应该写 `/blog/img/图片名.jpg`
- 文件名不要有中文和空格

### Q4：想删除一篇文章？

直接把 `_posts` 文件夹里对应的 `.md` 文件删除，然后推送（git push）。

### Q5：不小心把博客搞坏了怎么办？

别慌！GitHub 会保存你的每一次修改记录。

打开 `https://github.com/dalihaif/blog`，点 `Settings` → `Pages` → 看看有没有错误提示。

或者告诉我，我来帮你修复。

## 第八部分：进阶——自己动手改样式

如果你不满足于只是写文章，想让博客变得**独一无二**，可以改样式。

### 8.1 改颜色

博客的颜色定义在 `E:\my-web\blog\css\hux-blog.css` 文件里。

用记事本打开，搜索 `#` 开头的颜色代码，比如：

```
#0085A1  ← 这是蓝色
#777777   ← 这是灰色
#FFFFFF   ← 这是白色
#000000   ← 这是黑色
```

想换颜色，把这些代码改成你想要的颜色就行。

> 🎨 不知道颜色代码？搜索"颜色代码生成器"，点一下就能拿到。

### 8.2 改字体

还是在 `hux-blog.css` 里，搜索 `font-family`，就能看到当前用的字体。

把字体名字改成你想用的，比如：

```css
font-family: "微软雅黑", "PingFang SC", sans-serif;
```

### 8.3 添加自定义HTML模块

如果你想在博客上加入一些特殊功能（比如音乐播放器、访客统计、二维码等等），可以这样做：

1. 去网上找相应的 HTML 代码（比如"网易云音乐外链生成器"）
2. 打开 `_layouts\post.html` 文件
3. 把代码粘贴到你想要的位置
4. 保存发布
