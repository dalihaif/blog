## 开篇：一个让你不舒服的实验

先做个实验。

我给AI同一句话，两种写法，你看结果差距有多大。

**写法A**（大多数人这么写）：

> 写一篇关于咖啡的博文。

**写法B**（读完这篇文章后你会这么写）：

> 你是一位在云南咖啡庄园工作了十二年的烘焙师，同时给《三联生活周刊》写专栏。现在写一篇1500字的博文，主题是"为什么深烘和浅烘之争是个伪命题"。
>
> 要求：用第一人称，像跟老朋友聊天一样；开头不要用"咖啡是"这种万能句式，从一个具体的场景切入——比如你上周在杯测桌上闻到的那个味道；中间要有至少两个真实细节（产区海拔、烘焙曲线数据、或者你手上的烫伤疤）；结尾不要总结，留一个开放性的画面。
>
> 参考风格：汪曾祺写吃的、Paul Kalanithi写当医生的笔触。

你猜结果怎样？

写法A给你的是一篇 Wikipedia 的扩写版——正确但无趣，像学生作文。写法B给你的是一篇你真的想读完的文章——有温度、有细节、有观点。

**同一台AI，同一个话题。差别只在提示词。**

这篇文章不是教你背100个"提示词模板"——那种东西三天就过时了。我要给你的是一套**思维操作系统**：理解了底层原理，你自己就能生成无数个精准的提示词。

## 一、第一性原理：AI到底是什么？提示词到底在做什么？

### 1.1 别再把AI当搜索引擎

大多数人对AI的理解停留在"高级搜索引擎"：输入关键词，它给我答案。这个心智模型是错的。

搜索引擎的逻辑是**检索**——在海量网页里找到最匹配的那一个。你给的关键词越短越好，因为搜索引擎自己会猜你想找什么。

AI的逻辑完全不同。AI是一个**概率预测引擎**。它不"找"答案，它"生成"答案——一个字一个字地预测下一个最可能出现的字。

### 1.2 提示词的本质：约束概率空间

想象AI的脑子里有一片无边无际的雾，雾里藏着所有可能的文字组合。你说"写一篇关于咖啡的博文"，雾并没有散多少——关于咖啡的博文可以是一万种东西：可以是学术论文、可以是小红书种草文、可以是咖啡馆老板的创业日记。

你每多给一个约束条件，雾就散一片，概率空间就缩小一圈。当概率空间足够小的时候，AI预测出来的文字就从"泛泛而谈"变成了"精准命中"。

**这就是提示词的第一性原理：**

> **提示词不是命令，是约束。你约束得越精确，AI的概率空间越小，输出质量越高。**

一个差的提示词，是在一片大雾里让AI随便走。一个好的提示词，是给AI画了一条跑道——它只能在跑道上跑，但跑道上的风景由你决定。

### 1.3 为什么"写详细点"不管用

很多人意识到提示词要详细之后，会犯另一个错误：堆砌形容词。

> "请写一篇非常详细的、生动的、有趣的、引人入胜的、富有感染力的关于咖啡的博文……"

这就像跟厨师说"做得好吃一点、好看一点、有创意一点"——厨师听了也不知道做什么。因为"好吃""好看""有创意"全是**评价标准**，不是**操作指令**。

厨师需要的是："用云南小粒咖啡豆，中度烘焙，做成手冲，用日式陶瓷杯盛，配一块伯爵茶味的饼干摆盘。"

AI也一样。它需要的不是形容词，而是**具体的约束条件**：用什么视角写、写多少字、给谁看、开头怎么切入、结尾怎么收、参考谁的风格、必须包含什么、必须避免什么。

## 二、升维框架：从0维到5维的提示词进化

理解了第一性原理，接下来给你一套实操框架。我把提示词的进化分成六个维度，从0维到5维。每升一维，输出质量就上一个台阶。

### 0维：关键词（大多数人停在这里）

```
写一篇关于咖啡的博文
```

**结果**：AI给你一篇百科全书式的文章。正确，但像塑料花——没有生命。

**问题**：概率空间太大。AI只能取"最大公约数"——写一篇谁看都觉得"还行"但谁都不会转发的东西。

### 1维：主题 + 篇幅

```
写一篇1500字关于咖啡的博文
```

**结果**：稍微好一点，至少有了篇幅控制。但仍然是"通稿"。

**升维动作**：加了"篇幅"这个约束。

### 2维：受众 + 风格 + 语气

```
写一篇1500字的咖啡博文，面向刚入门的咖啡爱好者，
语气轻松口语化，像朋友聊天，不要用专业术语
```

**结果**：开始有"人味"了。AI知道要避开术语、要用大白话，读起来不再像说明书。

**升维动作**：加了"受众"和"语气"两个约束。AI知道在为谁写了。

### 3维：结构 + 素材 + 排除项

```
写一篇1500字的咖啡博文，面向刚入门的咖啡爱好者，语气轻松口语化。

结构要求：
- 开头用一个具体场景切入（不要用"咖啡是"开头）
- 中间对比三种冲泡方式（手冲、法压壶、摩卡壶），每种讲一个优缺点
- 结尾不要总结，用一个生活化的画面收

素材要求：
- 提到云南咖啡豆
- 包含一个具体的冲泡参数（水温、粉水比）

排除项：
- 不要写历史起源
- 不要写"适量饮用有益健康"
- 不要用"总之""综上所述"
```

**结果**：这篇文章开始像"一个人写的"了。有结构、有细节、有取舍。不是面面俱到的通稿，而是有重点的好文章。

**升维动作**：加了"结构脚手架""必须包含""必须排除"三组约束。AI知道该往哪走、该避开什么了。

### 4维：角色 + 思考过程 + 迭代

```
你是一位在云南咖啡庄园工作了十二年的烘焙师，
同时给《三联生活周刊》写专栏。

现在写一篇1500字的博文，主题是"为什么深烘和浅烘之争是个伪命题"。

结构要求：
- 开头从一个具体的杯测场景切入
- 中间用你的亲身经历讲两个故事
- 结尾留一个开放性画面

在写之前，请先：
1. 列出3个可能的切入角度，告诉我你选哪个、为什么
2. 写出大纲，标出每个部分的字数分配
3. 等我确认大纲后再写正文
```

**结果**：这不是在"生成文章"了，这是在"协作创作"。AI先思考、再规划、再写，每一步你都能参与。

**升维动作**：加了"角色身份""思考步骤""分阶段交付"三个约束。AI从"一次性输出"变成了"协作伙伴"。

### 5维：独特视角 + 情感共鸣 + 意外性

```
你是一位在云南咖啡庄园工作了十二年的烘焙师，
同时给《三联生活周刊》写专栏。

写一篇1500字的博文，主题是"为什么深烘和浅烘之争是个伪命题"。

但我不想要一篇"正确"的文章。我想要一篇让人读完以后
愣三秒钟、然后转发给朋友的文章。

做到这一点的方式：
- 找一个反常识的切入点——不是"各有千秋"这种正确的废话，
  而是一个真正让人意外的角度
- 写一个你自己都不确定答案的困惑——真诚的困惑比确定的结论更打动人
- 在文章中间放一个"顿悟时刻"——读者跟着你的思路走，
  突然在某个瞬间想通了一件事
- 结尾不要给答案，给一个画面——让读者自己悟

参考：汪曾祺写食物时的"不评好坏、只写感受"，
Paul Kalanithi《当呼吸化为空气》里的"在专业中保持脆弱"

禁止：
- 禁止出现"其实""说白了""归根结底"
- 禁止用排比句
- 禁止在结尾升华主题
```

**结果**：这篇文章会让你忘了它是AI写的。因为它有了最难量化的东西——**灵魂**。

**升维动作**：加了"反常识切入""真诚困惑""顿悟时刻""画面收尾"四个**情感层面的约束**，以及"禁止排比句""禁止结尾升华"两个**美学层面的约束**。

### 维度速查表

| 维度 | 核心约束 | 效果 | 类比 |
|------|----------|------|------|
| 0维 | 关键词 | 通稿 | 给厨师说"做饭" |
| 1维 | +篇幅 | 有控制 | 给厨师说"做两人份" |
| 2维 | +受众+语气 | 有人味 | 给厨师说"给小孩做，别太辣" |
| 3维 | +结构+素材+排除 | 像人写的 | 给厨师说"三菜一汤，有鱼有青菜，别放香菜" |
| 4维 | +角色+思考+迭代 | 像协作 | 让厨师先报菜单你选，选完再做 |
| 5维 | +反常识+情感+美学 | 有灵魂 | 让厨师做一道你没吃过但会记住的菜 |

**大多数人停在0-2维。读完这篇文章后，你应该能稳定地在3-4维操作，关键文章冲5维。**

## 三、五大核心诀窍（附模板和对比）

### 诀窍一：角色锚定法 —— 让AI"成为"某个人

**原理**：AI的概率空间里有无数种"说话方式"。你给它一个角色，它就从无数种方式中锁定到一种——那个角色的方式。

**差提示词**：
> 写一篇关于摄影入门的文章。

**好提示词**：
> 你是一位教了十五年摄影的老摄影师，带过上千个学生。你最受不了的就是新手一上来就问"买什么相机"。现在写一篇摄影入门指南，用你的口吻——有点毒舌但句句在理，像跟徒弟说话一样。

**为什么有效**：角色不只是"身份"，它自带语气、自带态度、自带经验视角。"教了十五年的老摄影师"和"摄影博主"写出来的东西完全不同——前者有教学生的经验、有对常见错误的了解、有一种过来人的从容。

**角色锚定模板**：
```
你是一位 [具体身份]，在 [具体领域] 工作了 [具体时间]，
[一个体现你性格/经历的细节]。
你的说话风格是 [具体描述，不是"幽默"而是"像XX一样"]。
```

**关键**：身份要具体（不是"专家"而是"教了十五年摄影的老法师"），性格要有细节（不是"幽默"而是"有点毒舌但句句在理"）。

### 诀窍二：边界约束法 —— 告诉AI"不要做什么"比"要做什么"更重要

**原理**：AI的默认行为是"面面俱到"——什么都提一点，什么都不深入。你不告诉它"别做什么"，它就会把所有"相关内容"都塞进来。

**差提示词**：
> 写一篇关于早起的心得体会。

**好提示词**：
> 写一篇关于早起的心得体会，800字。

> 不要：
> - 不要引用名人名言
> - 不要写"一日之计在于晨"
> - 不要列12345条建议
> - 不要在结尾说"坚持就是胜利"
> - 不要用"其实""说实话"开头

> 要：
> - 用一个你（虚构的）昨天早上六点的具体场景开头
> - 中间讲一个早起"失败"的故事——不是所有早起都是美好的
> - 结尾是一个安静的、没有说教的画面

**为什么有效**："不要"清单比"要"清单更有效，因为AI的默认输出里80%是"正确的废话"。砍掉废话，剩下的就是真东西。

**边界约束模板**：
```
禁止使用以下表达：[列出3-5个你不想看到的词/句式]
禁止出现以下内容：[列出2-3个你不想看到的话题]
必须包含以下元素：[列出2-3个你想要的具体细节]
```

### 诀窍三：结构脚手架法 —— 给AI一个骨架

**原理**：AI写长文最大的问题是"散"——写着写着就跑题，或者结构混乱。给它一个明确的骨架，它就能在骨架上长肉。

**差提示词**：
> 写一篇关于读书的文章，2000字。

**好提示词**：
> 写一篇关于读书的文章，2000字，结构如下：

> 第一部分（300字）：场景切入——你在某个具体场景下读书的画面，不要说"读书很重要"
> 第二部分（600字）：一个反常识的观点——比如"快速阅读是个陷阱"，用你的经历论证
> 第三部分（600字）：一个具体的方法——不是泛泛而谈，而是具体到"拿起书的第一分钟做什么"
> 第四部分（500字）：一个故事——你（虚构的）因为读某本书而改变了某个决定的真实感故事
> 每个部分之间用自然过渡连接，不要用"首先""其次""最后"

**为什么有效**：结构脚手架解决了三个问题：①跑题（每段有明确主题）②比例失衡（每段有字数分配）③衔接生硬（给了过渡方式）。

**结构脚手架模板**：
```
文章分为X部分，共XXXX字：
- 第一部分（XXX字）：[具体内容方向] + [开头方式要求]
- 第二部分（XXX字）：[具体内容方向] + [核心论点/故事]
- ...
- 结尾部分（XXX字）：[收尾方式要求]

各部分之间用 [过渡方式] 连接。
```

### 诀窍四：示例引导法 —— 给AI一个"靶子"

**原理**：AI最擅长的事情之一是"模仿"。你给它一个风格样本，它就能捕捉到那个风格的精髓——比任何形容词都精准。

**差提示词**：
> 写一篇关于秋天的散文，风格优美。

**好提示词**：
> 写一篇关于秋天的散文，600字。

> 风格参考——以下是我想要的感觉：

> "秋天是安静地来的。不是一夜之间满城黄叶那种来法，而是你某天早上出门，觉得空气里有什么不一样了，但又说不上来。后来你看见路边第一片落下的叶子，才确认：哦，秋天来了。"

> 注意这种风格的特点：
> - 不渲染、不煽情，克制
> - 用日常细节而非华丽辞藻
> - 有一种"迟到的发现感"

**为什么有效**：形容词是模糊的——"优美"可以是朱自清的优美，也可以是余秋雨的优美，完全不同。但一个50字的具体示例是精确的——AI能从示例中提取出节奏、用词、句式、情绪，精准复刻。

**示例引导模板**：
```
风格参考——以下是我想要的感觉：
[贴一段50-100字的范文，可以是别人的，也可以是你自己写的半成品]

注意这种风格的特点：
- [节奏特点]
- [用词特点]  
- [情绪特点]
```

### 诀窍五：迭代精炼法 —— 好内容是改出来的，不是一次生成的

**原理**：即使是最完美的提示词，一次性生成也很难达到最佳。真正高质量的内容，是"生成→反馈→修改→再生成"的迭代过程。

**第一次提示词**（生成初稿）：
> [你的完整提示词]

**拿到初稿后，第二步**（不要重新生成，要在初稿基础上改）：
> 初稿整体不错，但有几个问题需要调整：
> 1. 开头太长了，前三段砍到一段，直接进场景
> 2. 第二部分的论据不够有力，加一个具体的数据或案例
> 3. "其实""说实话""总的来说"这些词全部删掉
> 4. 结尾太"鸡汤"了，把最后两段合成一段，不要给结论
> 5. 整体压缩300字，去掉所有"正确的废话"

**第三步**（精修）：
> 现在最后过一遍：
> - 检查每句话是否都有信息量，没有信息量的句子删掉
> - 检查有没有连续两句用了相同的句式，有就改掉
> - 把三个最平淡的动词换成更精准的动词
> - 确认结尾最后一个画面是否足够有力

**为什么有效**：AI不像人类会"自我审视"。你让它一次性写完，它不会回头检查。但如果你分步骤给它反馈，每一步它都能精准执行。迭代三次的效果，是一次性生成的三倍以上。

**迭代心法**：
```
第一轮：要骨架——先要大纲，确认结构对了再写
第二轮：要血肉——写正文，先求完整不求完美
第三轮：要灵魂——针对性修改，砍废话、换动词、调节奏
第四轮：要打磨——逐句检查，删到不能再删
```

## 四、通用提示词模板库（直接复制使用）

以下四个模板覆盖了最常见的写作场景。复制 → 填空 → 用。

### 模板A：博文写作

```
你是一位 [具体身份]，在 [领域] 有 [年数] 经验。

写一篇 [字数] 字的博文，主题是 [主题]。

受众：[具体描述目标读者]
语气：[具体描述，如"像跟同事聊天，不是上课"]

结构：
- 开头：[开头方式，如"用一个上周发生的事切入"]
- 中间：[核心内容方向，如"讲三个教训，每个配一个真实案例"]
- 结尾：[收尾方式，如"不要总结，留一个问题给读者"]

必须包含：
- [具体细节1]
- [具体细节2]

禁止：
- [禁用词/句式1]
- [禁用词/句式2]
- [禁止出现的内容]

风格参考：[参考人物或贴一段范文]
```

### 模板B：故事创作

```
你是一位擅长 [类型] 的小说家，你的风格接近 [参考作家]。

写一个 [字数] 字的 [短篇/章节]，核心冲突是 [一句话描述冲突]。

主角：[名字]，[年龄]，[职业]，[一个性格矛盾点——比如"勇敢但怕黑"]
场景：[时间][地点][氛围]
关键转折：[在故事三分之二处发生什么]

要求：
- 开头第一句话必须是一个动作，不要背景介绍
- 对话不超过总篇幅的三分之一
- 用感官细节（至少涉及三种感官：视觉、听觉、嗅觉、触觉、味觉）
- 不要解释角色的心理活动，用行为暗示
- 结尾要有一个"回旋"——前面埋的某个细节在结尾产生了新含义

禁止：
- 不要用"突然""忽然"
- 不要在紧张时刻放慢节奏去描写环境
- 不要让角色"想到"——让角色"做"
```

### 模板C：心得体会

```
你是一位 [身份]，刚经历了 [具体事件/项目/阅读]。

写一篇 [字数] 字的心得体会。

核心要求：
- 不要写成"总结报告"——不要"通过这次XX我学到了XX"
- 要写成"复盘反思"——最大的收获是什么？最大的失误是什么？
  如果重来一次会怎么做？

结构：
- 第一段：直接说最核心的一个感悟（一句话）
- 第二段：这个感悟是怎么来的——讲具体发生了什么
- 第三段：为什么这个感悟重要——它改变了什么
- 第四段：我还不确定的事情——真诚地列出困惑
- 第五段：下一步打算做什么

语气：诚实、不装、有自我批评

禁止：
- 禁止"受益匪浅""感触颇深""收获满满"
- 禁止排比句
- 禁止在结尾说"未来我会继续努力"
```

### 模板D：深度长文（万字级）

```
你是一位 [身份]，要写一篇 [字数] 字的深度长文，主题是 [主题]。

这是一个需要深度思考的主题，请按以下步骤进行：

第一步：先列出5个可能的切入角度，每个用一句话描述。
       然后告诉我你推荐哪个角度，为什么。

第二步：用推荐角度写出详细大纲。
       - 每个部分标注预计字数
       - 标注每个部分的核心论点
       - 标注需要补充的素材/数据

第三步：等我确认大纲后，逐部分撰写。
       每写完一部分，停下来让我反馈，再写下一部分。

全局要求：
- 每个观点必须有论据支撑（数据、案例、引用）
- 每800字左右设一个"节奏变化点"——切换叙事视角或插入案例
- 避免"论文腔"——用讲故事的逻辑而非论证的逻辑
- 重要概念第一次出现时用一句话解释清楚
```

## 五、迭代心法：好内容是"聊"出来的

### 5.1 不要追求一次成功

很多人用AI的方式是：写一个提示词 → 生成 → 不满意 → 换一个提示词 → 生成 → 还是不满意 → 放弃。

这就像写文章只写第一稿，写完不改就发表。没有人第一稿就是完美的。

**正确的心态是**：把AI当成一个写作搭档。你给它方向，它写初稿；你给反馈，它改；你再给反馈，它再改。通常2-3轮就能达到很好的效果。

### 5.2 反馈要具体，不要笼统

**差的反馈**：
> "写得不太好，能不能更好一点？"

AI不知道"好"是什么。你得告诉它具体哪里不好、怎么改。

**好的反馈**：
> 第三段的例子太泛了，换一个具体的故事——比如你自己（虚构的）上个月遇到的类似情况。另外，第五段和第六段之间过渡太硬了，加一句承上启下的话。最后一段最后一句删掉，停在倒数第二句的画面上。

### 5.3 "手术刀"式修改法

不要每次都让AI重写全文。大多数时候只需要局部修改：

- "只改开头，其他不动"
- "第二段的论据换成更有说服力的"
- "把全文的'非常'全部删掉"
- "结尾加一个反转"

这种精准修改既快又不会破坏已经写好的部分。

### 5.4 一个完整的迭代流程示例

```
第一轮：要大纲
  → "先给我三个切入角度，我选一个"
  → AI给三个角度，你选第二个

第二轮：要大纲
  → "用第二个角度，写出详细大纲，每段标注字数"
  → AI给大纲，你改了两处

第三轮：要初稿
  → "按这个大纲写初稿"
  → AI写完，整体不错但有些问题

第四轮：要修改
  → "开头砍一半、第二段加数据、结尾换一种收法"
  → AI改完，好了很多

第五轮：要打磨
  → "逐句检查，删掉所有没有信息量的句子，
     把三个平淡的动词换成精准的动词"
  → AI打磨完，定稿
```

五轮迭代，从"还行"到"惊艳"。

## 六、六个致命误区

### 误区一："提示词越长越好"

**错**。提示词不是越长越好，是越**精确**越好。500字的提示词如果80%是废话，不如100字的精准约束。

判断标准：每个字是否都在缩小概率空间？如果某句话删掉，AI的输出不会变——那这句话就是废话。

### 误区二："给AI自由发挥的空间"

**错**。AI不需要创作自由。你给的空间越大，它越往"最大公约数"靠拢——写出来的东西谁都不讨厌但谁都不喜欢。

真正的好内容来自**约束下的创造力**——就像俳句的5-7-5音律限制没有扼杀创造力，反而催生了世界上最精炼的诗。

### 误区三："先写完再说"

**错**。最浪费时间的做法是让AI先写2000字，然后你发现方向完全不对，全部重来。

正确做法：**先花5分钟敲定大纲**，确认方向对了再写。大纲阶段调整方向成本是5分钟，正文阶段调整方向成本是30分钟。

### 误区四："用形容词控制风格"

**错**。"幽默""优美""深刻"这些词对AI来说是模糊的——它理解的"幽默"可能是讲笑话，而你想要的是冷幽默。

用**具体示例**控制风格。贴一段50字的范文比说100个形容词都管用。

### 误区五："AI写的直接用"

**错**。AI写的东西再好，也需要你过一遍。不是改错别字——AI很少写错别字。而是改"AI味"：

- AI喜欢用排比句 → 保留一组，删掉其他
- AI喜欢在结尾升华 → 砍掉最后一段
- AI喜欢用"其实""说白了""归根结底" → 全部删掉
- AI喜欢面面俱到 → 删掉最弱的那个观点
- AI的过渡句很生硬 → 删掉过渡句，直接换段

### 误区六："一个提示词走天下"

**错**。不同类型的内容需要不同的提示词策略：

- **信息类文章**（教程、科普）→ 重结构、重准确性、重排除项
- **故事类内容**（小说、叙事）→ 重角色、重冲突、重感官细节
- **观点类文章**（评论、心得）→ 重角度、重论据、重反常识
- **情感类内容**（散文、回忆）→ 重画面、重克制、重留白

## 七、终极心法：提示词是思考的镜像

说了这么多技巧，最后说一个最重要的事。

**提示词的质量，本质上是你思考质量的外化。**

一个提示词写得模糊，不是因为你不会用AI，而是因为你还没想清楚：
- 你到底要写什么？
- 给谁看？
- 想传达什么？
- 什么风格？
- 怎么开头、怎么结尾？

当你把这些想清楚了，提示词自然就写好了。当你没想清楚，再多的技巧也救不了。

所以，写提示词之前，先问自己五个问题：

1. **这篇文章的核心信息是什么？**（一句话能说清的那种）
2. **读者读完以后应该有什么感受？**（不是"学到了知识"，而是"原来如此"或"我也要试试"）
3. **如果只能保留一个段落，保留哪个？**（这决定了文章的核心）
4. **这篇文章和网上同类文章的区别在哪？**（如果没有区别，就别写了）
5. **开头第一句话怎么写才能让人停下滑动？**

想清楚这五个问题，你的提示词就已经打败了95%的人。

## 八、如果只记住三件事

如果这篇文章你只能记住三件事：

**第一**：提示词不是命令，是约束。你约束得越精确，AI的概率空间越小，输出质量越高。每个字都该在缩小概率空间。

**第二**：好内容不是一次生成的，是迭代的。先要大纲、再要初稿、再要修改、再要打磨。四轮迭代的效果是单次生成的四倍。

**第三**：提示词是思考的镜像。提示词写得模糊是因为你没想清楚。先把五个问题想明白，提示词自然就好写了。

## 附：30分钟创作流程速查

```
0-5分钟  ｜想清楚五个问题（核心信息/读者感受/核心段落/差异化/开头）
         ｜
5-10分钟 ｜写提示词（角色+受众+结构+约束+风格参考）
         ｜
10-15分钟｜要大纲，确认方向
         ｜
15-25分钟｜要初稿 → 给具体反馈 → 修改
         ｜
25-30分钟｜打磨：删废话、换动词、砍结尾升华
         ｜
30分钟   ｜定稿，通读一遍，改掉"AI味"
```
